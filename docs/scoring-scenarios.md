# 採点シナリオ詳細

[採点方法の概要](./scoring.md) を補足し、**実際に scoring-tool が行う URL・操作・待機条件**をまとめたものです。実装の正は `scoring-tool/src/calculate.ts` および `scoring-tool/src/scoring/calculate_*.ts` です。

## 採点の共通前提

| 項目 | 内容 |
|------|------|
| 初期化 | 計測開始前に `POST /api/v1/initialize` が呼ばれる（必須） |
| 表示→操作の閾値 | 「ページの表示」9件の合計が **300 点未満**のとき、「ページの操作」5件は**実行されず 0 点** |
| 計測間隔 | 各ターゲットの計測のあと **10 秒**待機（サーバー負荷緩和） |
| ビューポート | 幅 1920 × 高さ 1080、デスクトップ想定（`formFactor: desktop`） |
| Lighthouse | `throttlingMethod: simulate`、対象 audit は下記「指標」参照 |

### 部分一致で特定の計測だけ実行する

```bash
cd scoring-tool && pnpm start --applicationUrl <URL> --targetName "投稿"
```

`--targetName` は計測名の**部分一致**です。利用可能な名前は `pnpm start --applicationUrl <URL> --targetName`（値なし）で一覧表示されます。

## ページの操作（各 50 点 × 5 = 250 点）

Lighthouse **User Flow** の `startTimespan` / `endTimespan` で囲んだ区間の **TBT × 25 + INP × 25** をシナリオスコアとします。  
操作は主に **Playwright**、計測は **Puppeteer + Lighthouse**（同一セッション連携）です。

以下、「**計測区間**」は timespan 内の操作のみを指します。

### 1. ユーザーフロー: ユーザー登録 → サインアウト → サインイン

| 段階 | 内容 |
|------|------|
| 事前 | `GET /not-found` でページ表示（timespan 外） |
| 計測区間 | サインイン →「初めての方はこちら」で新規登録へ |
| 入力（新規登録） | ユーザー名・名前: `superultrahypermiracleromantic`、パスワード: `superultra_hyper_miracle_romantic`（文字入力は概ね 10ms 間隔） |
| | 「登録する」→「マイページ」リンクが表示されるまで（タイムアウト 10s） |
| | 「アカウントメニュー」→「サインアウト」→「サインイン」ボタンが再表示されるまで |
| | 再度サインイン（上記ユーザー名・パスワード）→「マイページ」まで |

### 2. ユーザーフロー: DM送信

| 段階 | 内容 |
|------|------|
| 事前 | `GET /not-found` → サインイン（`o6yq16leo` / `wsh-2026`）→「DM」で `/dm` へ（timespan 外） |
| 計測区間 | 「新しくDMを始める」→ 相手ユーザー名 `g63iaxn5c` →「DMを開始」→ URL が `**/dm/*` になるまで |
| | 入力欄（名前「内容」）に複数行入力（`Shift+Enter` で改行）。1通目末尾「パフォーマンス改善のアドバイスをお願いします！」→ `Enter` で送信 |
| | 該当テキストを含む `li` が表示されるまで最大 30s |
| | 2通目（「LCPの改善方法を具体的に教えてください。」を含む）を同様に送信・待機 |

### 3. ユーザーフロー: 検索 → 結果表示

| 段階 | 内容 |
|------|------|
| 事前 | `GET /search` を表示 |
| 計測区間 | 検索テキストボックス（ラベルに `検索 (例: キーワード since:…` を含む）にクエリ入力。1回目は `アニメ since:2026-01-06` + `0`×20 + `x`（例: `アニメ since:2026-01-0600000000000000000000x`） |
| | 2回目は入力クリアのうえ `アニメ since:2026-01-06` + `0`×10 + `x until:2026-01-06` + `0`×10 + `x` |
| | 「検索」クリック → 見出しが正規表現 `/「アニメ/` に一致するまで最大 120s |
| | 入力クリア後、日付範囲付きの別クエリ（`since:` / `until:` 付き、`0` 連結長が異なる）でもう一度検索・同様に見出し待ち |

### 4. ユーザーフロー: Crok AIチャット

| 段階 | 内容 |
|------|------|
| 事前 | `GET /not-found` → サインイン（`o6yq16leo` / `wsh-2026`）。成功確認は「**Crok**」リンクの表示（マイページではない） |
| | 「Crok」で `/crok` へ |
| 計測区間 | プレースホルダ「メッセージを入力...」に `TypeScriptのtemplate literal typeとは何ですか` →「送信」 |
| | `role="status"` かつ名前「応答中」まで最大 60s |
| | 見出し「第六章：最終疾走と到達」まで最大 120s |
| | 続けて `ReactのuseTransitionの使い方の例を教えてください` を入力するまで（**2通目の送信ボタンは押さない**。ここで timespan 終了） |

Crok の **SSE プロトコルおよび画面に必要な情報の渡し方**はレギュレーションで固定です。計測は上記 UI 待機に依存します。

### 5. ユーザーフロー: 投稿

| 段階 | 内容 |
|------|------|
| 事前 | `GET /` → サインイン（`o6yq16leo` / `wsh-2026`）（timespan 外） |
| 計測区間 | **テキストのみ**: 「投稿する」→ ダイアログ「新規投稿」→「いまなにしてる？」に宮沢賢治『注文の多い料理店』冒頭の一節を入力 → 投稿 → タイムラインの `article` に当該全文が表示されるまで |
| | **画像**: 同様にモーダル → 短文 + `画像を添付` にファイル設定 |
| | **動画**: `動画を添付` にファイル設定 → テキスト・「動画プレイヤー」ボタン等の出現を待つ |
| | **音声**: `音声を添付` にファイル設定 → テキスト・「シャイニングスター」「魔王魂」等の表示を待つ |

**添付ファイルのパス（リポジトリルート基準）**

scoring-tool は `scoring-tool/src/scoring/calculate_post_flow_action.ts` 内で、リポジトリルートを `../../..` で解決し、次を参照します。

| 種別 | パス |
|------|------|
| 画像 | `docs/assets/analoguma.tiff` |
| 動画 | `docs/assets/pixabay_326739_kanenori_himejijo.mkv` |
| 音声 | `docs/assets/maoudamashii_shining_star.wav` |

**ローカルで投稿シナリオのみ試す場合**、scoring-tool をリポジトリ構成のまま実行し、上記ファイルが存在することを確認してください。
